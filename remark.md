# 一、产品文档
## 1.1 产品定位
一款面向个人开发者/职场人群的**本地私有化部署工作记录工具**，主打「极简录入、智能总结」，解决个人工作记录零散、周报手动整理耗时的痛点。无需登录，打开浏览器即可使用；数据本地MySQL存储，安全可控；AI周报模块无缝对接已有接口，彻底解放周报撰写效率。
- 核心价值：个人工作记录轻量化管理，零学习成本，本地部署无数据泄露风险；
- 目标用户：个人开发者、职场办公人员（单用户使用场景）；
- 产品形态：B/S架构（浏览器访问，本地运行），无需客户端安装。

## 1.2 核心产品功能（单用户精简版）
### ✅ 核心模块1：每日工作记录（核心基础功能）
专为单用户设计，极简交互，快速完成工作内容/问题记录。
1. 快速录入：支持「标题+内容」极简模式，内容区支持富文本（换行、加粗、分段），满足问题现象、解决方案、工作成果等记录需求；
2. 分类标签：内置标签（问题记录、需求开发、会议纪要、待办事项），支持手动选择，便于后续筛选查询；
3. 记录属性：每条记录自动生成「创建时间、更新时间」，支持编辑、删除、修改，历史记录永久保存（本地数据库）；
4. 快捷查询：支持按「日期范围、标签、关键词」筛选记录，快速定位历史工作内容；
5. 单用户适配：无需用户隔离，所有记录均为个人数据，无需担心多用户权限问题。
6. 支持图片上传，并读取文字内容，作为快速记录的内容。

### ✅ 核心模块2：AI智能周报总结（核心增值功能）
完全对接已有AI接口，无需自研AI能力，核心逻辑为「数据聚合+接口调用+结果展示」。
1. 核心逻辑：根据选择的日期范围（如本周一至周日），自动聚合该时间段内所有工作记录，结构化拼接为文本入参，调用AI接口生成周报；
2. 极简操作：选择起止日期 → 点击「生成周报」→ 3-5秒展示结果，无需额外配置；
3. 结果处理：支持对AI生成的周报进行「二次编辑、一键复制、导出为TXT」；
4. 周报维度：内容包含本周完成工作、问题及解决方案、工作成果、下周计划（维度由AI接口定义，工具仅做数据透传）。

### ✅ 基础辅助功能
1. 数据管理：支持记录分页展示、批量删除、按时间倒序排列；
2. 数据安全：本地MySQL存储，支持手动备份数据库，无云端上传风险；
3. 轻量化适配：页面无冗余功能，加载速度快，支持快捷键（Ctrl+S保存记录）。

## 1.3 产品核心特性
1. 零门槛使用：无需登录注册，打开浏览器即可记录，学习成本为零；
2. 部署简单：本地Python+MySQL环境即可运行，无需服务器/网络特殊配置；
3. 数据可控：所有记录本地存储，隐私安全有保障；
4. 灵活扩展：AI接口可自由替换，后续可按需添加功能；
5. 高效便捷：录入一条记录仅需10秒，周报生成一键完成。

## 1.4 产品核心使用流程（单用户闭环）
```
启动本地服务 → 浏览器访问工具 → 有日历形式展示整理记录清空，并支持快捷选择日期-【每日记录】录入工作内容/问题
→ 工作记录：支持按日期形式查看，支持二次编辑，删除
→ 每周按需：选择日期范围 → 点击【生成周报】→ 获取AI总结结果
→ 编辑/复制/导出周报 → 历史记录随时查询/修改
```

# 二、技术文档（单用户精简版）
## 2.1 技术栈总览（无冗余，适配单用户）
### ✅ 后端核心技术
- 主语言：Python 3.8+
- Web框架：Flask 2.3+（轻量化，适合单用户场景，开发效率高）
- 核心依赖库：pymysql（MySQL连接）、requests（对接AI接口）、python-dotenv（配置管理）
- 核心能力：记录CRUD、数据聚合、AI接口对接

### ✅ 前端核心技术
- 核心：原生HTML5 + CSS3 + JavaScript（无框架依赖）
- 样式适配：Bootstrap 5.x（CDN引入，响应式布局）
- 富文本编辑：TinyMCE（CDN引入，支持基础格式排版）
- 核心能力：页面渲染、表单提交、周报结果展示

### ✅ 数据库
- MySQL 5.7/8.0（本地部署，稳定可靠，适合存储个人结构化数据）

### ✅ 第三方接口
- AI周报接口：直接对接已有接口，仅需配置接口地址和入参格式。

## 2.2 系统架构（单用户极简架构）
```
【数据持久层】MySQL数据库 → 存储个人工作记录
        ↑↓
【后端服务层】Python+Flask → 核心业务逻辑：记录CRUD、数据聚合、AI接口对接
        ↑↓
【前端视图层】HTML+CSS+JS → 页面展示、用户交互
        ↑↓
【第三方接口】已有AI周报接口 → 接收数据并返回周报总结
```
架构优势：层级清晰，无冗余模块，单用户使用无性能压力，维护成本极低。

## 2.3 数据库设计（取消用户表，单表极简设计）
### 设计原则
单用户场景下仅需**1张核心业务表**，无冗余字段，兼顾性能和易用性。
> 数据库建议命名：`work_record_db`

### ✅ 核心表：工作记录表 `work_record`
```sql
CREATE TABLE `work_record` (
  `id` INT PRIMARY KEY AUTO_INCREMENT COMMENT '记录ID，主键',
  `title` VARCHAR(200) NOT NULL COMMENT '工作记录标题',
  `content` TEXT NOT NULL COMMENT '工作记录内容（富文本）',
  `tag` VARCHAR(50) DEFAULT '其他' COMMENT '标签：问题记录、需求开发、会议纪要、待办事项',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  INDEX idx_create_time (`create_time`),
  INDEX idx_tag (`tag`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='个人每日工作记录表';
```
**设计说明**：
- 保留核心索引（创建时间、标签），提升查询效率；
- `content`字段为TEXT类型，支持存储长文本内容。

## 2.5 部署与运行步骤（单用户极简版）
### 环境准备
1. 安装Python 3.8+和MySQL 5.7+/8.0；
2. 安装依赖：`pip install -r requirements.txt`；
3. 创建MySQL数据库`work_record_db`，执行上述`work_record`表的建表SQL。

### 配置修改
1. 打开后端代码，修改`DB_CONFIG`中的MySQL密码为自己的数据库密码；
2. 修改`AI_REPORT_CONFIG`中的3个配置项，对接已有AI接口。

### 启动运行
1. 运行Flask服务：`python app.py`；
2. 打开浏览器，访问地址：`http://localhost:5000`；
3. 直接使用记录和周报功能，无需登录。
